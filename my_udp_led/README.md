# RK端UDP通信程序

## 功能说明

本程序实现了RK端与FPGA的UDP通信，主要包含以下功能：

1. **接收FPGA测试数据**：接收FPGA发送的20字节测试数据（全为"w"字符）
2. **控制FPGA LED灯**：通过UDP数据包控制FPGA端的LED灯（LED2-LED8）

## 硬件连接

- **FPGA IP地址**：192.168.1.11
- **FPGA端口**：8080
- **RK IP地址**：192.168.1.105
- **RK端口**：8080

## LED映射关系

| LED编号 | 对应位 | 十六进制值 |
|--------|-------|-----------|
| LED2   | bit6  | 0x40      |
| LED3   | bit5  | 0x20      |
| LED4   | bit4  | 0x10      |
| LED5   | bit3  | 0x08      |
| LED6   | bit2  | 0x04      |
| LED7   | bit1  | 0x02      |
| LED8   | bit0  | 0x01      |

**注意**：LED1是心跳灯，由FPGA内部控制，无法通过本程序控制。

## 运行环境

- Python 3.x
- 网络连接正常
- FPGA和RK在同一局域网内

## 使用方法

1. **修改IP地址**：如果实际网络环境与默认配置不同，请修改`udp_communication.py`文件中的IP地址和端口号。

2. **运行程序**：
   ```bash
   cd rk_test
   python udp_communication.py
   ```

3. **控制LED灯**：在命令行中输入以下命令：
   - `on <LED编号>`：打开指定LED灯（例如：`on LED8`）
   - `off <LED编号>`：关闭指定LED灯（例如：`off LED8`）
   - `all on`：打开所有LED灯（LED2-LED8）
   - `all off`：关闭所有LED灯（LED2-LED8）
   - `status`：显示LED映射关系和当前状态
   - `exit`：退出程序

4. **查看接收数据**：程序会自动接收并显示FPGA发送的测试数据。

## 注意事项

1. 确保FPGA和RK在同一局域网内，且IP地址配置正确。
2. 程序需要以管理员权限运行，以绑定UDP端口。
3. 如果防火墙阻止UDP通信，请确保允许UDP端口8080的通信。
4. LED1是心跳灯，由FPGA内部控制，会周期性闪烁。
